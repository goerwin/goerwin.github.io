This project consists of multiple web applications (payments, customer service, etc.) that use a micro-frontend strategy to communicate and share state.

Most of the micro-frontend apps use client-side React, Redux Toolkit, and Material UI. I worked on the initial setup using Webpack's Module Federation, creating the Shell application that hosted all micro-frontends from other teams.

During this process, I participated in many meetings with technical leads across the company, discussing whether the micro-frontend approach would work for all teams. We also considered monolithic approaches.

On the back-end side, I improved SEO for some landing pages by moving them from client-side to server-side rendering with Next.js.

My responsibilities were:

* Discuss and research pros/cons of Micro-frontend vs Monolithic architecture
* Investigate and create the micro-frontend approach for React applications
* Implement the micro-frontend approach using Webpack’s module federation
* Implement features using TypeScript, React, Next and Redux
* Participate in code review processes for multiple teams
* Create unit tests and integration tests using React Testing Library
* Attend refinement meetings, daily standups, demo meetings
* Propose new features and modification/elimination of existing ones
* Work with design team using Figma for collaboration
* Work on the Profile Management app using React, Material UI, Redux Toolkit, Jest, and React Testing Library
